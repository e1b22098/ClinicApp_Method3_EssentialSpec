# アプリケーション再起動手順

## システム全体の再起動は不要です

データベースの変更を反映させるためには、**Spring Bootアプリケーションを再起動するだけで十分**です。
OS全体を再起動する必要はありません。

## アプリケーションの再起動方法

### 方法1: アプリケーションが起動中の場合

1. **現在実行中のアプリケーションを停止**
   - ターミナルで `Ctrl + C` を押す
   - または、アプリケーションが実行されているウィンドウを閉じる

2. **再度起動**
   ```powershell
   java -jar target/appointment-1.0.0.jar
   ```
   または
   ```powershell
   mvn spring-boot:run
   ```

### 方法2: アプリケーションが停止している場合

そのまま起動すれば、最新のデータベース状態が読み込まれます：

```powershell
java -jar target/appointment-1.0.0.jar
```

## 確認事項

アプリケーションを再起動した後：

1. **管理者ログインを試す**
   - ID: `admin`
   - パスワード: `admin`

2. **エラーログを確認**
   - アプリケーション起動時にエラーメッセージが出ていないか確認
   - データベース接続エラーが出ていないか確認

## トラブルシューティング

### まだログインできない場合

1. **データベースの内容を再確認**
   ```sql
   USE clinic_booking_db;
   SELECT admin_login_id, name, LEFT(password, 30) as password_preview 
   FROM administrators 
   WHERE admin_login_id = 'admin';
   ```

2. **アプリケーションのログを確認**
   - 起動時のログにエラーが出ていないか確認
   - ログイン試行時のエラーメッセージを確認

3. **ブラウザのキャッシュをクリア**
   - 古いセッション情報が残っている可能性があります

## まとめ

✅ **アプリケーションを再起動するだけでOK**  
❌ OS全体の再起動は不要  

アプリケーションを再起動してから、再度ログインを試してください！
