# 実行時のトラブルシューティング

## ClassNotFoundException が発生する場合

### 解決方法1: クリーンビルドを実行

```powershell
mvn clean install -DskipTests
```

その後、再度実行：

```powershell
mvn spring-boot:run
```

### 解決方法2: IDEから直接実行

EclipseやIntelliJ IDEAなどのIDEを使用している場合：

1. `ClinicAppointmentApplication.java` を開く
2. `main`メソッドを右クリック
3. "Run As" → "Java Application" を選択

### 解決方法3: 実行可能JARを作成して実行

```powershell
# JARファイルを作成
mvn clean package -DskipTests

# JARファイルを実行
java -jar target/appointment-1.0.0.jar
```

### 解決方法4: 日本語パス名の問題を回避

プロジェクトを日本語が含まれないパスに移動する：

```powershell
# 例: C:\projects\ClinicApp に移動
# その後、mvn spring-boot:run を実行
```

## その他のエラー

### ポート8080が使用中

`application.yml`でポートを変更：

```yaml
server:
  port: 8081  # 他のポートに変更
```

### データベース接続エラー

1. MySQLが起動しているか確認
2. `application.yml`の接続情報が正しいか確認
3. データベース`clinic_booking_db`が作成されているか確認
